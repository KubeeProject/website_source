이런 기능을 만들어 보았습니다

Kubee-Project에서 구현한 클라우드 인프라 자동화 기능입니다

K3s 자동화 및 무중단 배포

Ansible Playbook을 통해 배포 과정을 자동화하고, K3s의 Rolling Update 전략을 결합하여 가동 중단 시간(Downtime) 0초를 실현했습니다. Readiness Probe를 설정하여 신규 Pod의 정상 동작 확인 후 교체되도록 구성, 안정적인 서비스 전환을 보장합니다.

지능형 Auto Scaling

Terraform의 Target Tracking 정책을 활용하여 CPU 사용률 50%를 일정하게 유지하는 지능형 리소스 최적화를 구현했습니다. 스트레스 테스트를 통해 트래픽 급증 시 서버가 자동으로 확장되는 유연성을 증명했습니다.

운영 시각화 및 모니터링

오픈소스 모니터링 솔루션을 구축하여 인프라 비용을 절감하고, K3s 클러스터 내 Pod 상태 및 리소스(CPU/Mem) 사용량을 실시간으로 시각화했습니다. Grafana 대시보드를 통해 팀 전체가 서비스 상태를 한눈에 파악할 수 있는 데이터 기반의 운영 환경을 조성했습니다.

CI/CD 자동화의 완성

GitHub Actions와 Terraform을 연동하여 인프라 변경부터 배포까지 전 과정을 자동화했습니다. 코드 한 줄로 인프라가 배포되는 '자동화의 끝'을 실현하여 개발 생산성을 극대화했습니다.

기능 시연

부하 테스트 시각화 CI/CD 자동화

팀 Kubee 소개

Kubee는 Kubernetes와 Bee(꿀벌)의 합성어입니다.

Kubernetes의 'Kube'와 꿀벌 'Bee'가 만났습니다.
벌집 속 꿀벌들이 정교하게 협동하듯, 클라우드 인프라의 자동화와 최적화를 설계합니다.
취업이라는 꿀을 향해 나아가는 준비된 엔지니어들이 모여,
기술적으로 빈틈없는 완벽한 3-Tier 아키텍처를 구축했습니다.

사용 기술

AWS (ALB, ASG, EC2) Terraform (IaC) Ansible K3s (Kubernetes) GitHub Actions Prometheus & Grafana Rolling Update Target Tracking Linux (Ubuntu)
